## Change control policy

### What requires Change Control

| Change type | Requires Change Control? |
|---|---|
| Risk limit modification (07_risk_safety.md) | Yes |
| Gate threshold modification (08_promotion_gates.md) | Yes |
| Feature set modification after holdout run | Yes |
| Model estimator change | Yes |
| Instrument or timeframe change | Yes |
| Training window change | Yes |
| Cost model assumptions change | Yes |
| Regime gate bounds change during live phase | Yes |
| Document corrections (typos, formatting) | No |
| Experiment log additions | No |
| New experiment prior to holdout | No |

### Change Control process

1. Describe the proposed change and rationale in this document.
2. Identify all impacted documents and update them in the same change.
3. Get approval from Trading + AI Engineering.
4. Increment version number in affected documents.
5. Add a "Change Rationale" section to any document with a substantive change.
